# Contributing

Thanks for improving these practice materials.

## Useful Contributions

- Fix typos or unclear explanations.
- Report environment setup problems.
- Add notes for Windows, macOS, or Linux.
- Improve exercises, starter code, or reference solutions.
- Add small datasets with clear sources and licenses.

## Contribution Flow

1. Open an issue describing the problem or improvement.
2. Fork the repository.
3. Create a branch.
4. Make a focused change.
5. Open a pull request.

## Practice Material Standard

Every chapter practice should include:

- Learning goals
- Setup requirements
- Starter files
- Step-by-step exercises
- Validation command or expected output
- Reference solution
- Common mistakes

## License Notes

Only contribute materials you have the right to share publicly.
