# 习题

## 基础（4 题）

1. **图像加载与模糊检测**：使用 OpenCV 加载一张交通场景图像，计算其拉普拉斯方差（Laplacian variance）作为模糊度指标。设定阈值，将方差低于阈值的图像标记为"模糊"。简述该方法的原理与局限性。

2. **IoU 计算**：给定两个边界框 `[x1, y1, x2, y2]`，编写函数计算它们的交并比（Intersection over Union）。用以下三组数据验证：(a) 完全重叠；(b) 部分重叠；(c) 完全不重叠。写出每种情况的 IoU 值。

3. **数据增广**：列举至少 5 种适用于交通场景的数据增广方法，并说明每种方法在什么条件下不应使用（例如极端旋转可能导致标签语义失效）。解释为什么垂直翻转通常不适合固定交通摄像头场景。

4. **NMS 原理**：给定一组检测框及其置信度分数，手工执行非极大值抑制（NMS）过程（IoU 阈值取 0.5），写出每一步保留和剔除的框。检测框如下：

   | 框编号 | 坐标 [x1,y1,x2,y2] | 置信度 |
   | --- | --- | --- |
   | A | [10,10,60,60] | 0.95 |
   | B | [12,12,58,58] | 0.85 |
   | C | [15,14,65,64] | 0.70 |
   | D | [100,100,150,150] | 0.90 |
   | E | [105,105,155,155] | 0.60 |

## 进阶（4 题）

1. **Precision-Recall 与 AP 计算**：给定 10 个检测结果（按置信度降序排列）及其是否为真正例（TP）或假正例（FP），绘制 Precision-Recall 曲线，并计算该类别的平均精度（AP）。使用全点插值法。

   检测结果序列：TP, TP, FP, TP, FP, TP, FP, FP, TP, FP。数据集中共有 6 个真实目标。

2. **mAP 计算**：假设三个类别的 AP 分别为：车辆 0.82、行人 0.65、自行车 0.71。计算 mAP@0.5。讨论在交通场景中，为什么行人类别的 AP 通常低于车辆类别。

3. **MOTA 与 IDF1 计算**：给定一段视频的跟踪结果，其中共有 20 帧出现目标。统计如下：总漏检数 5，总误检数 3，总 ID 切换数 2。计算 MOTA。若该段视频中 IDTP=45, IDFP=8, IDFN=10，计算 IDF1。讨论 ID Switch 对流量统计的影响。

4. **计数线逻辑与冷却期**：设计一条虚拟计数线（如画面中一条水平线 y=300），编写函数判断车辆是否从上往下穿越该线。输入为车辆中心坐标的时间序列 `[(x1,y1), (x2,y2), ...]`，输出为穿越次数。实现冷却期机制（穿越后 N 帧内不再计数），讨论为什么需要冷却期。

## 挑战（3 题）

1. **完整检测评价流程**：实现一个完整的检测评价模块，输入为真实标注和检测结果（均含类别、边界框、置信度），输出为每个类别的 AP 和整体 mAP。要求：支持多类别、支持 IoU 阈值可调、使用全点插值法。使用合成数据验证正确性。

2. **综合案例：路口视频流量统计管线**：基于 `simulate_video_pipeline` 函数，设计一个从视频到流量统计的完整管线框架。系统包含：视频帧提取、目标检测、NMS、多目标跟踪、计数线穿越检测、按时间窗汇总流量。画出系统流程图，写出关键模块的伪代码。讨论：(1) 若检测 mAP 较高但流量统计误差仍大，可能原因是什么？(2) 在夜间、雨天、遮挡等场景下各模块可能遇到的问题及应对策略。

3. **部署挑战分析**：选择一个真实路口监控场景，撰写一份 500-800 字的技术分析报告，讨论将实验室模型部署到实际路口时需要考虑的工程问题，包括但不限于：光照变化、天气影响、遮挡与拥挤、相机抖动与标定漂移、实时性要求、模型更新与维护、隐私合规。提出至少三项具体的技术措施。

## 思考题（对应教材习题）

1. 图像分类、目标检测、语义分割和多目标跟踪分别解决什么问题？它们在交通应用中各自适合什么场景？

2. 为什么仅有目标检测结果还不足以完成车辆流量统计？检测到跟踪再到计数，每一步引入的误差会如何传播？

3. 为什么 YOLO 类模型适合实时交通视频分析？其局限是什么？在什么情况下两阶段检测器更合适？

4. 设计一个交通视频速度估计方案，说明需要哪些相机标定信息（内参矩阵、外参矩阵、畸变系数），为什么透视变换是关键步骤？
