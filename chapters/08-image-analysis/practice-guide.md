## 3. 图像加载与质量控制

### 3.1 图像读取

使用 OpenCV 读取图像：

```python
import cv2

img = cv2.imread("traffic.jpg")
if img is None:
    raise FileNotFoundError("图像加载失败")
# OpenCV 默认 BGR 格式，转为 RGB
img_rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
```

### 3.2 模糊检测（拉普拉斯方差法）

原理：清晰图像的拉普拉斯变换响应值大（边缘多），模糊图像的响应值小（边缘少）。计算拉普拉斯变换结果的方差作为模糊度指标。

```python
def compute_blur_score(img_gray: np.ndarray) -> float:
    """计算图像的拉普拉斯方差，值越小越模糊"""
    return cv2.Laplacian(img_gray, cv2.CV_64F).var()
```

典型阈值：

- 方差 < 50：严重模糊，建议剔除或重新采集。
- 50-100：轻度模糊，可用于训练但标注质量可能下降。
- \> 100：通常认为清晰。

局限性：

- 对噪声敏感，高噪声图像可能获得较高的拉普拉斯方差。
- 只能衡量全局模糊，无法检测局部模糊（如运动模糊只影响部分区域）。
- 阈值依赖场景，不同摄像头和场景需要调整。

### 3.3 其他质量控制

- **亮度检查**：计算灰度图均值，过暗（< 40）或过亮（> 220）的图像可能不可用。
- **遮挡率检查**：如果使用了语义分割标注，可计算被遮挡类别的像素占比。
- **分辨率检查**：确认图像分辨率满足模型输入要求。

## 4. 数据增强

交通场景数据增强需要兼顾多样性和标签一致性。

### 4.1 推荐增强方法

| 方法 | 说明 | 注意事项 |
| --- | --- | --- |
| 水平翻转 | 镜像翻转，适用于对称场景 | 右行交通场景翻转后车辆行驶方向反转，需注意语义一致性 |
| 随机裁剪 | 裁剪子区域并缩放到原始尺寸 | 裁剪比例不宜过大，否则可能裁掉关键目标 |
| 色彩抖动 | 调整亮度、对比度、饱和度 | 模拟不同光照和天气条件 |
| 随机旋转 | 小角度旋转（-15° ~ +15°） | 大角度旋转会导致边界框标注失效 |
| 高斯噪声 | 添加随机噪声 | 模拟低质量摄像头 |
| 运动模糊 | 沿运动方向施加模糊 | 模拟高速运动目标 |
| 随机擦除 | 随机遮挡部分区域 | 模拟遮挡情况 |
| Mosaic | 四张图拼接为一张 | YOLOv4+ 常用，丰富小目标上下文 |

### 4.2 边界框同步

所有空间变换（翻转、裁剪、旋转）必须同步调整边界框坐标。例如水平翻转时：

```python
def flip_bbox_horizontal(bbox, img_width):
    x1, y1, x2, y2 = bbox
    return [img_width - x2, y1, img_width - x1, y2]
```

## 5. IoU 与 NMS

### 5.1 IoU（交并比）

IoU 衡量两个边界框的重叠程度，是检测评价和 NMS 的核心指标。

```
IoU = 交集面积 / 并集面积
```

实现要点：

- 正确计算交集区域：`inter_x1 = max(a_x1, b_x1)` 等。
- 无交集时返回 0。
- 面积计算使用 `(x2 - x1) * (y2 - y1)`，注意坐标约定。

### 5.2 NMS（非极大值抑制）

NMS 用于去除同一目标上的重复检测框。

算法步骤：

1. 按置信度从高到低排序所有检测框。
2. 取置信度最高的框 A，加入输出列表。
3. 计算剩余框与 A 的 IoU，剔除 IoU > 阈值的框。
4. 重复步骤 2-3 直到所有框被处理。

参数选择：

- IoU 阈值通常取 0.5（较严格）到 0.7（较宽松）。
- 阈值过低会误删不同目标的框，过高则保留过多冗余检测。

## 6. 检测评价：Precision、Recall、AP、mAP

### 6.1 基本定义

| 指标 | 定义 |
| --- | --- |
| TP（真正例） | 检测正确且 IoU >= 阈值的目标 |
| FP（假正例） | 检测错误（IoU < 阈值或重复检测） |
| FN（假反例） | 未被检测到的真实目标 |
| Precision | TP / (TP + FP)，检测结果的准确率 |
| Recall | TP / (TP + FN)，真实目标的召回率 |

### 6.2 AP 计算

AP（Average Precision）是 Precision-Recall 曲线下的面积。

全点插值法步骤：

1. 按置信度降序排列检测结果。
2. 逐个计算累积 Precision 和 Recall。
3. 对每个 Recall 值 r，取 r' >= r 范围内的最大 Precision。
4. 计算 Precision-Recall 曲线下面积。

### 6.3 mAP

mAP 是所有类别 AP 的均值。不同 IoU 阈值对应不同的 mAP：

- mAP@0.5：IoU 阈值为 0.5 时的 mAP。
- mAP@0.5:0.95：IoU 阈值从 0.5 到 0.95（步长 0.05）的平均 mAP。

## 7. 跟踪评价：MOTA 与 IDF1

### 7.1 MOTA（Multiple Object Tracking Accuracy）

```
MOTA = 1 - (FN + FP + IDSW) / GT
```

其中 GT 为真实目标出现总次数，IDSW 为 ID 切换次数。

MOTA 衡量跟踪的整体检测质量，受漏检、误检和 ID 切换影响。取值范围 (-inf, 1]，越高越好。

### 7.2 IDF1（ID F1 Score）

```
IDF1 = 2 * IDTP / (2 * IDTP + IDFP + IDFN)
```

IDF1 基于身份匹配而非框匹配，更关注跟踪的一致性。取值范围 [0, 1]，越高越好。

### 7.3 MOTA 与 IDF1 的区别

- MOTA 侧重检测质量，对 ID 切换的惩罚相对较小。
- IDF1 侧重身份一致性，对 ID 切换更敏感。
- 两者互补，应同时报告。

## 8. 计数线逻辑与流量统计

### 8.1 基本原理

在画面中设定一条虚拟计数线（通常为水平线或垂直线），跟踪每个目标在连续帧之间的位置变化，判断目标是否穿越计数线。

### 8.2 穿越判定

对于目标在第 t 帧和第 t+1 帧的中心坐标 (x_t, y_t) 和 (x_{t+1}, y_{t+1})，水平计数线 y = y_line：

```
穿越条件：y_t < y_line 且 y_{t+1} >= y_line（从上往下穿越）
或      y_t >= y_line 且 y_{t+1} < y_line（从下往上穿越）
```

### 8.3 去重与边界情况

- **同一目标多次穿越**：设置穿越冷却期或记录每个 track_id 的穿越状态。
- **目标在计数线附近来回移动**：增加最小穿越距离约束。
- **目标被短暂遮挡后重新出现**：跟踪算法需要处理断连重连（如 DeepSORT 的重识别模块）。

### 8.4 方向统计

可分别统计上行和下行流量，用于不同方向的交通分析。

## 9. 相机标定与速度估计

### 9.1 像素坐标到世界坐标

监控摄像头的图像坐标与真实世界坐标之间的映射需要相机标定：

- **内参矩阵**：描述焦距、主点等相机内部参数。
- **外参矩阵**：描述相机在世界坐标系中的位置和朝向。
- **畸变系数**：描述镜头径向和切向畸变。

### 9.2 逆透视映射（IPM）

将透视图像转换为鸟瞰图（俯视图），使得图像中的距离与真实距离成正比。

前提条件：

- 已知相机标定参数。
- 假设地面为平面（路口场景基本满足）。

### 9.3 速度估计

通过跟踪目标在连续帧之间的位移，结合 IPM 后的像素-距离换算比例和帧率，估计目标速度：

```
速度 = 像素位移 * 距离换算系数 * 帧率
```

精度受以下因素影响：标定精度、目标遮挡、帧率、IPM 假设。

## 10. 真实部署挑战

### 10.1 光照变化

- 问题：白天/夜晚、逆光、阴影导致图像质量剧烈变化。
- 应对：数据增强（色彩抖动、伽马校正）；使用多时段训练数据；引入光照不变特征。

### 10.2 天气影响

- 问题：雨、雾、雪降低可见度，雨滴附着镜头。
- 应对：去雾算法（暗通道先验）；天气条件数据增强；部署防护罩。

### 10.3 遮挡与拥挤

- 问题：车辆互相遮挡，高峰期目标密集。
- 应对：3D 检测模型；利用时序信息（跟踪）；多摄像头融合。

### 10.4 相机抖动与标定漂移

- 问题：风吹或震动导致画面抖动，长期使用后标定参数变化。
- 应对：图像配准（特征点匹配）；定期自动标定检查。

### 10.5 实时性要求

- 问题：路口实时检测需要低延迟（< 100ms/帧）。
- 应对：模型压缩（剪枝、量化、知识蒸馏）；边缘计算设备部署；输入降分辨率。

### 10.6 模型更新与维护

- 问题：新车型出现、季节变化导致数据分布漂移。
- 应对：在线学习与持续监控；A/B 测试新模型；异常检测触发人工审核。

### 10.7 隐私合规

- 问题：路口视频包含人脸、车牌等敏感信息。
- 应对：人脸/车牌模糊化；本地处理不上传原始视频；遵守数据保护法规。
