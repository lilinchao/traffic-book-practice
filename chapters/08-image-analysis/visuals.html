01

图像预处理与质量控制

影像分析的第一步是保证输入图像质量。模糊检测和图像增广是最常用的预处理手段。

模糊检测

通过 Laplacian 方差区分清晰与模糊图像，方差越低越模糊。

数据增广

翻转、亮度调整、噪声注入和随机裁剪，扩充训练样本多样性。

02

目标检测评价

IoU 定义预测框与真实框的重叠度，NMS 去除冗余检测框，P-R 曲线和 AP 衡量检测质量。

IoU 示意

交并比（Intersection over Union）衡量预测框与真实框的重叠程度。

NMS 过程

非极大值抑制去除同一目标上的重复检测框，保留置信度最高的。

P-R 曲线

不同置信度阈值下的 Precision 和 Recall 权衡关系。

AP 插值计算

11 点插值法计算 Average Precision，平滑 P-R 曲线的波动。

IoU 阈值如何选择

通常 IoU ≥ 0.5 视为正确检测（COCO 标准使用 0.50:0.95 多个阈值）。阈值越高要求定位越精确，AP 值越低。

mAP 的含义

mAP（mean Average Precision）是所有类别 AP 的平均值，综合反映检测器在不同类别上的表现。mAP@0.5 使用 IoU=0.5 阈值。

03

目标跟踪与流量统计

目标跟踪为检测框分配跨帧 ID，计数线穿越逻辑统计流量。MOTA 衡量跟踪质量，IDF1 衡量身份保持一致性。

计数线穿越

目标中心点穿越预设计数线时触发计数，支持方向判别。

ID 切换示例

目标被遮挡后重新出现时可能分配新 ID，ID 切换是跟踪常见错误。

跟踪质量曲线

随着检测质量提升，MOTA 和 IDF1 均上升，IDF1 对身份一致性更敏感。

05

语义分割与实例分割

语义分割为每个像素分配类别标签，实例分割进一步区分同一类别中的不同对象。交通场景中两者各有适用场景。

语义分割 vs 实例分割

左：原始场景；中：语义分割（所有车辆同色）；右：实例分割（每辆车不同色）。

何时用分割而非检测

若只需统计车辆流量，边界框检测通常已足够；若需估计占用面积、精确道路边界或可行驶区域，则分割更有价值。分割标注成本高于检测。

实例分割的价值

拥堵排队场景中，语义分割只能知道哪些像素属于车辆，无法区分具体是哪一辆；实例分割为每辆车生成独立掩膜，可直接计数。

06

相机标定与速度估计

视频速度估计的核心难点是透视效应：同样的像素距离在近处和远处代表不同真实距离。逆透视映射（IPM）将透视图像转换为鸟瞰图，使像素距离与真实距离成正比。

透视效应与 IPM

左：透视视图中相同真实距离对应不同像素距离；右：IPM 后像素距离一致。

07

综合案例：视频到交通指标的管线

路口交通视频车辆检测与流量统计的完整管线：从视频采集到流量汇总和误差分析，每一步都可能引入误差。

完整管线流程

视频采集 → 帧抽取 → 车辆检测 → NMS 去重 → 多目标跟踪 → 计数线穿越 → 流量汇总 → 误差分析。

误差传播

检测漏检会导致流量低估，误检导致高估。跟踪 ID 切换可能导致重复计数。计数线位置不当可能导致误计。若 mAP 较高但流量误差仍大，应检查跟踪和计数线逻辑。

结果评价三层级

第一层（视觉算法层）：检测和跟踪是否准确？第二层（交通指标层）：流量和速度是否接近真值？第三层（管理应用层）：结果能否支持信号配时和拥堵预警？
