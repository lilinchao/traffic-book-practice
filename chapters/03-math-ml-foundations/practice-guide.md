## 2. 数值微分与交通速度变化率

### 2.1 导数的交通含义

交通速度时间序列 v(t) 的导数 dv/dt 表示速度在时刻 t 的瞬时变化率。在交通场景中：

- dv/dt > 0：速度正在上升，可能是拥堵正在消散。
- dv/dt < 0：速度正在下降，可能是拥堵正在形成。
- |dv/dt| 很大：速度急剧变化，可能是事故或信号灯切换。

### 2.2 数值微分方法

给定离散时间序列 v_0, v_1, ..., v_{n-1}，时间步长 h：

**前向差分**：

```text
dv/dt |_i  ≈  (v_{i+1} - v_i) / h
```

**中心差分**（更精确）：

```text
dv/dt |_i  ≈  (v_{i+1} - v_{i-1}) / (2h)
```

### 2.3 实践步骤

1. 生成模拟的 24 小时速度序列：早高峰下降、中午恢复、晚高峰再下降。
2. 分别用前向差分和中心差分计算速度变化率。
3. 找出变化率最大的时刻，解释发生了什么。
4. 比较两种差分方法的结果差异。

### 2.4 交通解读

变化率最大的时刻往往对应拥堵的起止点。交通管理部门可以根据变化率极值判断拥堵开始时间，提前启动管控措施。

## 3. 概率分布与交通事故计数

### 3.1 泊松分布

泊松分布常用于建模单位时间或单位空间内随机事件的发生次数。在交通中，路口事故数、路段违规数等计数数据天然适合泊松建模。

概率质量函数：

```text
P(X = k) = (lambda^k * e^{-lambda}) / k!
```

其中 lambda 是单位时间内的期望事件数。泊松分布要求均值等于方差（等离散）。

### 3.2 负二项分布

交通数据往往存在过度离散（方差 > 均值），此时泊松分布不再适用。负二项分布引入额外的离散参数 alpha，允许方差大于均值：

```text
Var(X) = mu + alpha * mu^2
```

alpha 越大，过度离散越严重。当 alpha -> 0 时，负二项分布退化为泊松分布。

### 3.3 实践步骤

1. 生成 lambda=5 的泊松随机样本 1000 个，绘制直方图并叠加理论 PMF。
2. 生成 r=5, p=0.5 的负二项随机样本 1000 个，同样绘图比较。
3. 用 `scipy.stats` 对同一组数据分别拟合泊松和负二项分布，比较拟合效果。
4. 用卡方检验判断哪个分布更适合。
5. 解释：如果数据存在过度离散，泊松拟合会出现什么问题？

### 3.4 交通解读

- 泊松分布适合"稀有独立事件"，如低事故率路口的日均事故数。
- 负二项分布适合"聚集性事件"，如存在未观测异质性的路段事故数（有些路段天生事故更多）。
- 选择正确的分布直接影响模型推断的可靠性。

## 4. 矩阵运算与速度矩阵

### 4.1 速度矩阵的结构

在交通检测网络中，数据通常组织为速度矩阵 V，其中：

- 行：时间片（如每小时一个时间片，24 行代表一天）。
- 列：检测器（如 10 个检测器，10 列代表 10 个位置）。

```text
V ∈ R^{T x D}
```

T 为时间片数，D 为检测器数。

### 4.2 基本矩阵运算及其交通含义

| 运算 | 数学表达 | 交通含义 |
| --- | --- | --- |
| 行均值 | v_bar_t = (1/D) * sum_d V_{t,d} | 每小时全网平均速度 |
| 列均值 | v_bar_d = (1/T) * sum_t V_{t,d} | 每个检测器的日平均速度 |
| 矩阵转置 | V^T | 交换时间和空间的视角 |
| 矩阵乘法 | V^T V | 检测器之间的速度相关性矩阵 |
| SVD 分解 | V = U S V^T | 将时空模式分解为基模式的叠加 |

### 4.3 SVD 与低秩近似

对速度矩阵做 SVD 分解后，保留前 k 个奇异值可以得到低秩近似：

```text
V_k = U_k S_k V_k^T
```

k 的交通含义：

- k=1：只用一个模式（全天平均模式）表示速度场，精度低但最简洁。
- k=2-3：捕捉早晚高峰等主要时空模式。
- k 较大：逐渐还原细节，包括个别检测器的异常波动。

### 4.4 实践步骤

1. 构造 24 x 10 的模拟速度矩阵，包含早高峰和晚高峰模式。
2. 计算行均值、列均值，绘制速度热力图。
3. 做 SVD 分解，观察前 3 个奇异值的大小。
4. 用前 k=1,2,3 个奇异值重建矩阵，计算 Frobenius 范数重建误差。
5. 观察每个奇异向量对应的时间模式和空间模式。

## 5. 梯度下降与线性回归

### 5.1 问题设定

给定交通特征矩阵 X（n 个样本 x p 个特征）和目标值 y（如交通流量），线性回归模型为：

```text
y_hat = X w + b
```

损失函数（MSE）：

```text
L(w, b) = (1/n) * sum_i (y_i - y_hat_i)^2
```

### 5.2 梯度下降算法

梯度下降通过迭代更新参数来最小化损失：

```text
w <- w - lr * dL/dw
b <- b - lr * dL/db
```

其中 lr 是学习率。梯度计算：

```text
dL/dw = -(2/n) * X^T (y - Xw - b)
dL/db = -(2/n) * sum(y - Xw - b)
```

### 5.3 学习率的影响

- lr 太小：收敛速度慢，需要很多轮迭代。
- lr 太大：损失震荡甚至发散，无法收敛。
- lr 合适：损失稳定下降，较快收敛。

### 5.4 实践步骤

1. 生成模拟交通数据：X 为时间特征（小时编码），y 为对应流量。
2. 初始化参数 w=0, b=0。
3. 分别用 lr=0.001, 0.01, 0.1 运行 200 轮梯度下降。
4. 记录每轮损失，绘制三条收敛曲线。
5. 用最终参数做预测，与 `numpy.linalg.lstsq` 的解析解比较。

## 6. 时间序列的数据划分

### 6.1 为什么不能随机打乱

交通数据是时间序列，样本之间存在时间相关性。如果随机打乱再划分：

- 训练集中可能包含未来数据，模型"偷看"了未来的信息。
- 测试集可能包含过去数据，评估结果过于乐观。
- 这种现象称为数据泄露（data leakage）。

### 6.2 时间序列划分方法

**简单时间划分**：按时间顺序，前 80% 作为训练集，后 20% 作为测试集。

```text
训练集：t = 0, 1, ..., 0.8 * n - 1
测试集：t = 0.8 * n, ..., n - 1
```

**滚动窗口划分**（进阶）：用固定长度窗口逐步向后滚动，更稳健但更复杂。

### 6.3 实践步骤

1. 生成 365 天的交通流量数据。
2. 分别用随机划分和时间划分切分数据。
3. 用相同模型分别训练和评估，比较两种划分下的测试集误差。
4. 观察随机划分下误差是否系统性偏低，解释原因。

## 7. 评价指标与交通含义

### 7.1 三种常用指标

给定真实值 y 和预测值 y_hat：

**MAE（平均绝对误差）**：

```text
MAE = (1/n) * sum_i |y_i - y_hat_i|
```

含义：平均每个预测偏离真实值多少个单位。对异常值不敏感。

**RMSE（均方根误差）**：

```text
RMSE = sqrt((1/n) * sum_i (y_i - y_hat_i)^2)
```

含义：对大误差惩罚更重。如果个别时间点预测严重偏差，RMSE 会显著增大。

**MAPE（平均绝对百分比误差）**：

```text
MAPE = (1/n) * sum_i |(y_i - y_hat_i) / y_i| * 100%
```

含义：相对误差的百分比。适合比较不同量级数据的预测精度。

### 7.2 交通场景解读

| 指标 | 低流量时段行为 | 高流量时段行为 | 交通解读 |
| --- | --- | --- | --- |
| MAE | 绝对偏差小，但不代表相对精度高 | 绝对偏差可能较大 | "平均差多少辆车" |
| RMSE | 对小偏差不敏感 | 对大偏差敏感，容易受突发拥堵影响 | "有没有严重预测失败的时段" |
| MAPE | y_i 接近 0 时可能趋于无穷，不适合低流量 | 相对稳定 | "百分比精度" |

### 7.3 实践步骤

1. 生成一组预测值和真实值，包含低流量和高流量时段。
2. 分别计算 MAE、RMSE、MAPE。
3. 刻意加入一个低流量时段的大百分比偏差，观察 MAPE 的变化。
4. 刻意加入一个高流量时段的大绝对偏差，观察 RMSE 的变化。
5. 写出选择指标的建议：什么场景用什么指标。

## 8. 综合实验

### 实验流程

```text
生成模拟数据 -> 特征工程 -> 时间序列划分 -> 梯度下降训练 -> 预测 -> 计算评价指标 -> 分析结果
```

### 实验要求

1. 模拟 730 天（2 年）的交通流量数据，包含趋势、周期性和噪声。
2. 特征工程：提取星期几、是否节假日、滞后 1 天流量等特征。
3. 按时间顺序划分：前 80% 训练，后 20% 测试。
4. 用梯度下降训练线性回归。
5. 计算 MAE、RMSE、MAPE，并分析每种指标的交通含义。
6. 对比训练集和测试集误差，判断是否存在过拟合。

### 过拟合的信号

- 训练集误差远低于测试集误差。
- 添加更多特征后训练集误差下降，但测试集误差上升。
- 解决方法：减少特征、增加正则化、增加数据量。
