## 基础（Basic）

1. **识别数据类型**：下表记录了某路口每天各小时的车辆通行数（行 = 日期 × 小时，列 = 路口编号、时间、通行数）。这属于哪种数据类型？请从面板数据、时间序列数据、空间数据、时空数据、图像数据中选择，并说明理由。

   | 日期 | 小时 | 路口编号 | 通行数 |
   | ---- | ---- | -------- | ------ |
   | 2024-01-01 | 08 | L001 | 1230 |
   | 2024-01-01 | 09 | L001 | 1180 |
   | 2024-01-01 | 08 | L002 | 890 |
   | 2024-01-01 | 09 | L002 | 920 |

2. **描述分析框架**：选择一个你关心的交通问题（例如"为什么早高峰某路段经常拥堵"），按照"问题-数据-方法-解释"四个步骤，写出你的分析思路。每步至少写一句话。

3. **搭建环境**：在你的电脑上完成以下操作，并在 `notes.md` 中记录每步的输出结果：
   - 运行 `python --version` 确认 Python 版本
   - 运行 `python starter/hello_ai.py` 确认脚本可执行
   - 列出当前已安装的与数据分析相关的包（可用 `pip list`）

4. **认识数据集目录**：阅读本书 `README.md` 或相关文档中提到的公开数据集列表，选择一个你感兴趣的数据集，写出它的名称、数据类型、以及你认为它适合研究的交通问题。

   - 列出可能需要的数据及来源

## 挑战（Challenge）

1. **公开数据集探索**：从以下数据集中选择一个，下载并完成初步探索：

   - PeMS（加州交通性能测量系统）：https://pems.dot.ca.gov/
   - NYC TLC Trip Record Data：https://www.nyc.gov/site/tlc/about/tlc-trip-record-data.page
   - Chicago Traffic Tracker：https://data.cityofchicago.org/

   要求：
   - 读取数据，输出行数、列数和各列的数据类型
   - 检查缺失值比例
   - 绘制一个简单的分布图或时间趋势图
   - 判断该数据属于哪种类型（面板/时间序列/空间/时空/图像），并写一段分析说明

2. **数据类型分类系统**：编写一个 Python 函数 `classify_data_type(df)`，输入一个 pandas DataFrame，自动判断该数据最可能属于哪种交通数据类型。判断逻辑至少考虑以下维度：
   - 是否有时间列
   - 是否有空间/地理列（经纬度、区域编码）
   - 是否有图像相关列（文件路径、像素矩阵）
   - 是否有多个个体/实体的重复观测

3. **完整分析框架实践**：选择一个交通公开数据集，按"问题-数据-方法-解释"框架完成一个完整的迷你分析：
   - 定义一个明确的分析问题
   - 加载和清洗数据
   - 选择并实现一种分析方法（可以是简单的统计汇总或可视化）
   - 写出分析结论和局限性

4. **数据目录编写**：为你选择的数据集编写一份"数据目录"文档，内容至少包括：
   - 数据集名称和来源
   - 数据采集方式与时间范围
   - 各字段名称、类型和含义
   - 已知的数据质量问题
   - 适用的分析场景推荐
