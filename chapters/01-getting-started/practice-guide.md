# 第 1 章实践讲义：交通数据挖掘绪论

本讲义面向学生上机实践使用，不作为书籍正文。目标是帮助学生理解交通数据挖掘的核心分析框架，识别常见交通数据类型，了解公开数据集资源，并完成分析环境搭建。

## 1. 实践目标

完成本实践后，你应该能够：

- 用"问题-数据-方法-解释"框架分析一个交通问题
- 从数据表中识别面板、时间序列、空间、时空、图像五种数据类型
- 阅读交通数据集目录并选择适合研究问题的数据
- 搭建并验证 Python 数据分析环境

## 2. "问题-数据-方法-解释"框架

交通数据挖掘遵循一个四步循环框架。每一步都有明确的目标和常见陷阱。

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  问题定义  │ ──▶ │  数据获取  │ ──▶ │  方法选择  │ ──▶ │  结果解释  │
│ Problem   │     │ Data     │     │ Method   │     │Interpret │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
      ▲                                                   │
      └───────────────────────────────────────────────────┘
                        反馈迭代
```

### 框架详解

| 步骤 | 核心任务 | 常见问题 |
| ---- | -------- | -------- |
| 问题定义 | 明确要回答什么问题、目标变量是什么 | 问题太宽泛、目标不可测量 |
| 数据获取 | 寻找合适的数据源、理解数据采集方式 | 数据与问题不匹配、忽略数据偏差 |
| 方法选择 | 根据数据类型和研究问题选择分析或建模方法 | 方法与数据类型不匹配、忽略模型假设 |
| 结果解释 | 将方法输出转化为有意义的交通洞察 | 过度解读相关性为因果、忽略不确定性 |

### 案例：城市路口拥堵分析

以"某城市主干道早高峰为什么经常拥堵"为例，演示框架的应用：

**第一步：问题定义**

- 问题：某城市主干道在工作日早高峰（7:00-9:00）频繁发生拥堵，原因是什么？
- 目标变量：路段通行时间或拥堵指数
- 分析粒度：按天按 15 分钟间隔

**第二步：数据获取**

- 需要的数据：
  - 交通流量数据（时间序列或时空数据）
  - 信号灯配时数据
  - 事故/施工记录
  - 天气数据（可能的干扰因素）
- 可能来源：城市交通管理平台、PeMS、气象局开放数据

**第三步：方法选择**

- 描述性分析：拥堵的时间分布和空间分布特征
- 相关性分析：拥堵与流量、天气、事故的关系
- 可能的建模方法：时间序列模型（如 ARIMA）、回归模型

**第四步：结果解释**

- 例如："当流量超过 1800 辆/小时且发生事故时，拥堵概率从 12% 上升到 65%"
- 注意：这是相关性描述，不是因果结论
- 需要进一步考虑混杂因素（如道路等级、时间段、季节等）

## 3. 交通数据类型识别

交通数据可以分为五种主要类型。识别数据类型是选择分析方法的前提。

### 3.1 面板数据（Panel Data）

**特征**：多个个体在多个时间点上的重复观测。

**交通样例**：

| 路段 ID | 日期 | 事故数 | 平均速度 |
| ------- | ---- | ------ | -------- |
| L001 | 2024-01-01 | 2 | 45.3 |
| L001 | 2024-01-02 | 0 | 52.1 |
| L002 | 2024-01-01 | 1 | 38.7 |
| L002 | 2024-01-02 | 3 | 41.0 |

**判断要点**：
- 有多个个体（路段、区域等）
- 有时间维度
- 同一个体在不同时间有多次观测

**适用方法**：固定效应模型、随机效应模型、面板回归

### 3.2 时间序列数据（Time Series Data）

**特征**：单个个体随时间变化的观测序列。

**交通样例**：

| 时间戳 | 流量（辆/5分钟） |
| ------- | ---------------- |
| 2024-01-01 08:00 | 230 |
| 2024-01-01 08:05 | 245 |
| 2024-01-01 08:10 | 258 |

**判断要点**：
- 只有一个个体（或把每个个体单独分析）
- 按时间顺序排列
- 关注时间依赖性（趋势、周期、自相关）

**适用方法**：ARIMA、指数平滑、LSTM

### 3.3 空间数据（Spatial Data）

**特征**：数据带有空间位置信息，关注空间分布和空间关系。

**交通样例**：

| 站点名称 | 经度 | 纬度 | PM2.5 浓度 |
| -------- | ---- | ---- | ---------- |
| 科技园站 | 113.95 | 22.53 | 35 |
| 深圳湾站 | 113.97 | 22.52 | 42 |

**判断要点**：
- 有经纬度或区域编码等空间标识
- 关注"在哪里"和"空间关系如何"
- 可能需要 GIS 工具处理

**适用方法**：空间自相关分析、克里金插值、空间回归

### 3.4 时空数据（Spatiotemporal Data）

**特征**：同时包含空间和时间维度，是最常见的交通数据类型。

**交通样例**：

| 出租车 ID | 时间戳 | 经度 | 纬度 | 速度 |
| --------- | ------ | ---- | ---- | ---- |
| T001 | 2024-01-01 08:00 | 113.95 | 22.53 | 35.2 |
| T001 | 2024-01-01 08:05 | 113.96 | 22.54 | 28.7 |
| T002 | 2024-01-01 08:00 | 114.05 | 22.55 | 40.1 |

**判断要点**：
- 同时有空间列和时间列
- 关注"何时在何地"的联合模式
- 数据量通常较大

**适用方法**：时空自相关、轨迹挖掘、时空图神经网络

### 3.5 图像数据（Image Data）

**特征**：以图像形式采集的交通数据。

**交通样例**：

| 图像文件 | 拍摄时间 | 摄像头 ID | 场景类型 |
| -------- | -------- | --------- | -------- |
| cam_001_20240101_0800.jpg | 2024-01-01 08:00 | CAM-001 | 路口 |
| cam_002_20240101_0800.jpg | 2024-01-01 08:00 | CAM-002 | 高速 |

**判断要点**：
- 原始数据是图片或视频帧
- 通常需要计算机视觉方法处理
- 可能从图像中提取结构化信息（如车辆计数）

**适用方法**：目标检测、语义分割、图像分类

### 3.6 数据类型速查表

| 数据类型 | 时间维度 | 空间维度 | 个体数 | 典型交通场景 |
| -------- | -------- | -------- | ------ | ------------ |
| 面板数据 | 有 | 可有 | 多个 | 多路段事故统计 |
| 时间序列 | 有 | 无 | 单个 | 单检测器流量 |
| 空间数据 | 无 | 有 | 多个 | 空气质量站点 |
| 时空数据 | 有 | 有 | 多个 | 出租车轨迹 |
| 图像数据 | 可有 | 可有 | -- | 路口监控截图 |

## 4. 如何阅读交通数据集目录

一个好的数据集目录应该包含以下信息。在实践开始前，先花时间阅读目录可以帮助你判断数据是否适合你的研究问题。

### 4.1 目录关键要素

| 要素 | 含义 | 为什么重要 |
| ---- | ---- | ---------- |
| 数据集名称 | 数据集的正式名称 | 便于检索和引用 |
| 数据来源 | 谁采集、如何采集 | 理解数据偏差和局限性 |
| 时间范围 | 数据覆盖的时间段 | 判断时效性和时间跨度 |
| 空间范围 | 数据覆盖的地理区域 | 判断是否适合研究区域 |
| 字段说明 | 每列的含义和类型 | 理解数据内容 |
| 更新频率 | 数据多久更新一次 | 判断是否能用于实时分析 |
| 许可协议 | 数据的使用限制 | 确保合规使用 |

### 4.2 阅读目录时的检查清单

- [ ] 数据的时间范围是否覆盖我需要的时段？
- [ ] 数据的空间范围是否覆盖我研究的区域？
- [ ] 数据的粒度（时间粒度、空间粒度）是否满足分析需求？
- [ ] 字段含义是否清晰？是否有数据字典？
- [ ] 数据质量是否有说明？已知问题有哪些？
- [ ] 数据许可是否允许学术使用？

## 5. 环境搭建清单

### 5.1 必需软件

| 软件 | 最低版本 | 用途 | 验证命令 |
| ---- | -------- | ---- | -------- |
| Python | 3.9+ | 编程语言 | `python --version` |
| pip | 最新 | 包管理 | `pip --version` |
| Git | 2.0+ | 版本控制 | `git --version` |

### 5.2 推荐安装的 Python 包

| 包名 | 用途 | 安装命令 |
| ---- | ---- | -------- |
| numpy | 数值计算 | `pip install numpy` |
| pandas | 数据处理 | `pip install pandas` |
| matplotlib | 绑图 | `pip install matplotlib` |
| seaborn | 统计可视化 | `pip install seaborn` |
| jupyter | 交互式编程 | `pip install jupyter` |
| scikit-learn | 机器学习 | `pip install scikit-learn` |

也可以一次性安装：

```bash
pip install numpy pandas matplotlib seaborn jupyter scikit-learn
```

### 5.3 验证步骤

```bash
# 1. 确认 Python 版本
python --version

# 2. 运行本章入门脚本
python starter/hello_ai.py

# 3. 确认关键包可用
python -c "import numpy; import pandas; import matplotlib; print('All key packages available.')"

# 4. 启动 Jupyter（后续章节将使用）
jupyter notebook
```

### 5.4 常见问题

| 问题 | 原因 | 解决方法 |
| ---- | ---- | -------- |
| `python` 命令找不到 | 未安装或未加入 PATH | 安装 Python 并配置环境变量 |
| `ModuleNotFoundError` | 包未安装 | 运行 `pip install <包名>` |
| Jupyter 无法启动 | 端口被占用 | 运行 `jupyter notebook --port 8889` |
| 中文显示乱码 | matplotlib 字体缺失 | 配置中文字体（见第 2 章说明） |
